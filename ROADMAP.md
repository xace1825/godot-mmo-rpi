# Fantasy Settlement MMO — Roadmap

Цель: сделать persistent-world settlement builder в духе Songs of Syx / RimWorld с MMO-элементами.

---

## Этап 0. Стабилизация (1–3 дня)

Готовность: Windows-клиент подключается и работает без явных багов.

- [x] Сервер на RPi, UDP 7777
- [x] Авто-реконнект клиента
- [x] Ресурсы стартуют на складе (500/500/500)
- [x] Ручной спавн жителей через SPAWN
- [x] Сброс мира через RESET
- [x] Лут виден на клетках (ground items)
- [x] Инфо-панели для складов и жителей
- [x] A* pathfinding (greedy + anti-backtracking)
- [x] Проверить камеру и сквозные кнопки в Windows-клиенте
- [x] Автосохранение и автозапуск сервера (cron watchdog)
- [x] Полный аудит кода (32 проблемы найдено, 22 исправлено)

Ключевые файлы: `main.gd`, `build_ui.gd`, `network.gd`, `game_state.gd`

## Этап 1. Playable Core (1–2 недели)

Готовность: первые 10 минут игры понятны и веселы.

- [x] Профессии: дровосек, каменщик, фермер, строитель
- [x] Производственная цепочка: дерево → доски, камень → блоки, еда → еда
- [x] Жители переносят ресурсы в склад
- [x] Стены / полы / двери строятся удобно
- [x] Базовые нужды жителей: еда и сон
- [x] Сохранение/загрузка мира без потери состояния
- [x] Визуально читаемая карта и UI
- [x] Production stations walkable (NPC may stand on station tile)
- [x] Haulers no longer stuck after eating/sleeping
- [x] State clobber eat→sleep fixed
- [x] Worker resets to idle when station destroyed
- [x] Resources dropped on ground instead of destroyed
- [x] recalculate_rooms throttled (2s batch)
- [x] Network: no redundant full-state broadcasts

Ключевые файлы: `job_manager.gd`, `game_state.gd`, `planet_generator.gd`, `build_ui.gd`

## Текущий фокус

Этап 1 — playable core. Стабилизация и аудит пройдены; профессии, производственные цепочки и нужды работают. Следующий шаг — тестирование в Windows-клиенте и переход к Этапу 2.

## Конвенции проекта

- `TILE_SIZE = 32`, DawnLike 16→32 nearest
- Авторитет сервера на всем состоянии
- Ручной спавн жителей через SPAWN
- `Engine.time_scale = 1.0` (было 5.0 для тестов, исправлено)
- Все ресурсы живут на складах, не в глобальном пуле
- `.godot/` в `.gitignore`, клиент перегенерирует `.ctex`
- Pathfinding: greedy step + anti-backtracking (избегает возврата на предыдущий тайл)
- Watchdog cron job перезапускает сервер на RPi каждые 2 минуты при падении
